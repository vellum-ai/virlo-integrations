# Results Viewer app

Interactive viewer for Virlo Content Research Agent results, rendered in the
Vellum workspace panel. App ID: `plugins~virlo~virlo-results-viewer`.

This is a multi-file React/TSX app, built the way Vellum's plugin app runtime
expects (see https://www.vellum.ai/docs/extensibility/apps): TypeScript/TSX
source lives in `src/` and the bundler (esbuild) compiles it to a sibling
`dist/` on demand. `dist/` is generated and git-ignored. React is authored as
ordinary React and mapped to `preact/compat` internally by the bundler, so no
runtime dependency is declared here.

```
apps/virlo-results-viewer/
  src/
    index.html            HTML entry — loads the bundled main.js / main.css
    main.tsx              React root render + stylesheet import
    App.tsx               Top-level state machine: input → loading → error → results
    styles.css            All styling (CSS custom properties, responsive)
    types.ts              TypeScript shapes for the route payload
    lib/
      api.ts              fetchResults() — calls the plugin route
      format.ts           fmt() compact number formatting
    components/
      InputScreen.tsx     Paste-an-agent-UUID landing screen
      Loading.tsx         Spinner while fetching
      ErrorScreen.tsx     Error state with retry
      ResultsView.tsx     Header + tab bar + active tab
      StatsBar.tsx        Shared headline-stats row
      VideosTab.tsx       Top videos card grid
      OutliersTab.tsx     Creator outlier cards
      HashtagsTab.tsx     Ranked hashtags with animated bars
      SoundsTab.tsx       Rising sounds grid
  dist/                   Generated by the bundler (not tracked)
```

## Data flow

The app never touches the Virlo API directly. On load it reads an optional
`?agent_id=<uuid>` query parameter (auto-loading if present), then fetches
`GET /x/plugins/virlo/results?agent_id=<uuid>`. That route (`routes/results.ts`
at the plugin root) resolves the Virlo API key from the credential store and
aggregates all result endpoints — videos, creator outliers, hashtags, and
rising sounds — into a single JSON payload, which this app renders across four
tabs.
